Data sources

NVD API v2
National Vulnerability Database. Query by CPE or keyword to retrieve CVEs with CVSS scores and vector strings.
→ nvd.nist.gov/developers
EPSS — FIRST.org
Exploit Prediction Scoring System. Probability (0–1) that a CVE will be exploited in the next 30 days.
→ first.org/epss/api
CISA KEV
CISA Known Exploited Vulnerabilities catalog. CVEs here are confirmed as actively exploited in the wild.
→ cisa.gov/kev
Vulners API
Alternative vulnerability search engine with exploit references, often faster than NVD for PoC discovery.
→ docs.vulners.com

Scan methodology

01
Detect technology & version
Fetch HTTP response headers (Server, X-Powered-By) and HTML body. Apply regex fingerprints to identify stack (Apache, Nginx, PHP, WordPress…) and extract version numbers.
02
Normalize to CPE
Convert detected technology + version to a CPE 2.3 string: cpe:2.3:a:apache:http_server:2.4.49:*:*:*:*:*:*:* — the standard identifier used by NVD.
03
Query CVE databases
Hit NVD API v2 with CPE or keyword. Cross-reference with CISA KEV (confirmed exploited) and EPSS (exploitation probability). Flag CVEs with public PoC references.
04
Prioritize findings
Score each CVE: CISA KEV = critical priority · CVSS ≥ 9.0 = critical · EPSS > 0.5 = high exploitation risk. Sort results: KEV first, then CVSS descending.
05
Audit security headers
Check presence and quality of CSP, HSTS, X-Frame-Options, XCTO, Referrer-Policy, Permissions-Policy, COOP, CORP. Flag weak configurations (e.g. CSP with unsafe-inline).
06
Reproduce in controlled lab
For critical CVEs, spin up a matching Docker environment (docker compose up). Reproduce the vulnerability in isolation. Capture evidence (screenshots, curl output, Wireshark). Document the fix.
